<Title id="1" title="1. push" />

<Subtitle id="1.1" title="1.1 HEAD보다 이전의 커밋까지만 push해야 하는 경우" />

```
git push <Remote명> <commit SHA:Branch명>
```

<Subtitle id="1.2" title="1.2 로컬에 있는 사항을 강제로 force push해야 하는 경우" />

```
git push <Remote명> <Branch명> -f
```
```
git push <Remote명> +<Branch명>
```

- 단, 강제로 force push 할 때, 다른 사용자들과 공유되는 상황이라면 커밋들이 유실될 수 있다.
- 따라서, 로컬 저장소와 비교했을 때, 원격 저장소에 새로운 커밋이 추가되어 있는 경우
- 덮어쓰기 작업을 취소시키는 --force-width-lease 옵션을 사용하는 것이 더 안전한다.

```
git push <Remote명> <Branch명> --force-width-lease
```

<Subtitle id="1.3" title="1.3 현재 브랜치에서 다른 원격 브랜치로 push해야 하는 경우" />

- 현재 작업 중인 브랜치에 있는 커밋을 다른 원격 브랜치로 push해야하는 상황이 존재한다.
- 예를들어, 로컬 브랜치 이름을 master, 다른 원격 브랜치 이름을 feature/event라고 한다면 다음과 같은 명령어를 사용하면 된다.

```
git push origin master:feature/event
```

<Title id="2" title="2. cherry-pick" />

- 다른 브랜치에 있는 커밋을 현재 브랜치로 원하는 커밋을 가져 올 때 사용하면 된다.

```
git cherry-pick <commit SHA>
```

- 만약 여러 커밋을 한번에 가져오고 싶다면 다음과 같은 명령어를 사용하면 된다.

```
git cherry-pick <commit SHA> <commit SHA> <commit SHA> ...
```

- 단, cherry-pick 명령어를 사용하게 되면 원래 커밋의 해쉬값과 다르므로 주의해야 한다.


<Title id="3" title="3. rebase" />

```
git rebase -i <타켓 commit의 이전 commit SHA>
```
